App Info

Each message is encrypted/decrypted locally, on browser, by using the AES-256 algorithm in GCM mode.
Remote server is only used to store the encrypted message, so it can be later retrieved by a third person.
Sender has the ability to receive a delivery notification on his browser once his message is read by a recipient.
The encryption key which is used to encrypt/decrypt a message is locally created on user's browser and never gets exposed to server.
Every generated message is removed from server upon its retrieval. Also, unread messages are automatically destroyed 1 hour after their creation timestamp.
All of the above makes message creation, transmission and retrieval extremely secure, where data breach practically becomes impossible.

Encryption library Stanford Javascript Crypto Library (SJCL)
Official Web Page / Source code @ Github Encryption algorithm AES-256 in GCM mode with 128-bits tag Encryption key
  • Auto-generated at run-time (on browser) by using the SJCL library's provided features
  • Generated key length: 50 chars in base64/url-safe format
  • The generated key is not directly used for encryption/decryption, but instead, it is first processed by a key derivation function (PBKDF2 with 100,000 iterations)
 What data does server store? The server stores only the minimum information required from the encryption library to decrypt the message, based on the data that was originally used to encrypt it.
